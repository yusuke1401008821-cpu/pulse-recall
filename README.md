# Pulse Recall

ANKI に着想を得た、モバイル向けの暗記サポートアプリです。依存なしの静的 Web アプリをベースにしつつ、必要なときだけ Supabase を接続して共有デッキ化できる構成にしています。

## できること

- デッキ作成
- デッキ編集・削除
- カード追加
- カード編集・削除
- 表裏それぞれ最大4枚までの `画像付きカード`
- 画像の `IndexedDB` 保存と学習画面での拡大表示
- 初回 4 ステップガイド
- 今日の復習カード表示
- `忘れた / あいまい / 覚えた` に応じた次回復習間隔の調整
- `新規学習 / 学習中 / 定着レビュー / 再学習` の段階管理
- 回答前に次回目安が見える学習ボタン
- `復習 / 小テスト / 4択クイズ` の3つの学習モード
- `クイック追加` による連続カード作成
- `一括入力` による講義メモからの候補レビュー化
- `復習待ち / 直近ミス / あいまい多め / 伸び悩み` の弱点スマートリスト
- `高速学習` の `5分だけ回す / ミスだけ潰す / 4択で流す` 導線
- 小テスト / 4択での `ミスの即時再出題`
- デッキごとの `作成デフォルト` と `画像中心 / 文字中心` スタイル
- 直近7日間の学習履歴グラフと評価内訳
- `ホーム / 学習 / 作成 / ライブラリ / 設定` の5画面ナビ
- やりたいことから探せる `機能検索`
- ホームの `科目別デッキボード` と `学習 / 編集 / 資料追加` のクイック導線
- `作成` タブ内の `デッキ別編集ワークスペース`
- `作成` タブ内のモード別ミニガイド
- `設定` タブ内のアカウント保存、ガイド再表示、JSON バックアップ、初期化導線
- PDF / テキスト資料からのカード候補自動生成
- Gemini 無料枠を使った `AIで高精度生成`
- 候補プレビュー後のまとめてデッキ化
- 候補の一括選択 / 一括削除 / 一括タグ付け / 重複統合
- 過去問 PDF と講義スライド PDF の対応づけ
- 過去問参照の AI 根拠補強
- 設問ごとの関連スライド候補 / ページ番号 / 該当箇所の抜粋表示
- 学習タブでの AI 小テスト / AI 4択クイズ案
- ライブラリ内の保存済みカード検索
- ライブラリの `タグ / 分野 / 保存先 / 学習状態 / 文字検索` フィルタ
- ローカル検索結果を `新規デッキ` または `既存デッキ` にカード候補化
- JSON エクスポート / インポート
- 画像本体を含む JSON バックアップ / 復元
- Google / Apple / メールで使える `アカウント保存`
- `最新の自動バックアップ 1件 + 復元ポイント最大5件`
- 復元前の安全退避つき `この端末に復元`
- 共有リンクベースのグループ共有デッキ
- Supabase 未設定でも使える `共有リンクでローカル複製`
- 共有とアカウント保存で共通の Supabase ログイン
- owner 承認制の参加申請
- owner による `viewer / editor` 承認とロール変更
- 共有リンクの再発行、共有終了、メンバー退出
- ローカル複製共有 / 共同編集共有の使い分けガイド
- 共有デッキ内容と個人学習進捗の分離
- `localStorage` による端末内保存
- 画像本体の `IndexedDB` 分離保存
- PWA 用の manifest / service worker の同梱

## 起動方法

```bash
python3 -m http.server 4173
```

その後、ブラウザで [http://localhost:4173](http://localhost:4173) を開きます。

## Vercel 公開準備

このプロジェクトは HTML / CSS / JavaScript の静的サイトなので、Vercel では `Other` を選び、`Build Command` を空欄にするとそのまま配信できます。

公開時のおすすめ設定:

- Framework Preset: `Other`
- Build Command: 空欄
- Output Directory: `.` または未設定
- Root Directory: このプロジェクトのルート

公開後に検索へ載せやすくするために、以下は追加済みです。

- `robots.txt`
- PWA 用アイコン
- Open Graph / description 系メタタグ
- `vercel.json` による `sw.js` と manifest の更新向けヘッダー

## PDF 取り込みについて

- `作成` タブの `PDF 取り込み` から使えます
- PDF と `.txt` / `.md` のテキスト資料に対応します
- PDF の文字抽出はブラウザ上で行うため、最初の利用時はオンライン環境が必要です
- 画像だけのスキャンPDFは文字抽出できないことがあります
- `AIで高精度生成` を押すと、Gemini の無料枠で高精度生成を試します
- AI が未設定、無料枠上限、通信失敗のときは自動でローカル解析へ戻ります

## 過去問参照について

- `作成` タブの `過去問参照` から使えます
- 過去問 PDF と講義スライド PDF を端末内だけで解析し、設問ごとに近いスライド候補を表示します
- 表示されるのは `関連スライド候補 / ページ番号 / 該当箇所の抜粋 / 一致キーワード` です
- 完全一致ではなく関連度ベースなので、短い設問よりも選択肢付きの過去問の方が見つかりやすいです
- `AIで高精度補強` を押すと、ローカル候補の根拠説明を Gemini で補強します

## ライブラリ検索について

- `ライブラリ` タブから質問できます
- 保存済みカードだけを参照する安全モードです
- 外部AIや課金APIは使いません
- 追加したカードや PDF から作ったカードも、そのまま検索対象になります

## 学習モードについて

- `学習` タブの上段に `高速学習` があり、`5分 / 10分 / 20分` からすぐ始められます
- `5分だけ回す / ミスだけ潰す / 4択で流す` の3導線があり、条件に合うカード数から問題数を自動で決めます
- `高速学習` では、4択の正誤表示後に自動で次へ進む設定や、ミスの即時再出題を使えます
- `学習` タブで `復習 / 小テスト / 4択クイズ` を切り替えられます
- `小テスト` は記述式の思い出し確認で、答え確認後に `正解 / 惜しい / 不正解` を自己採点できます
- `4択クイズ` はカードの答えから選択肢を作り、正誤と解説をその場で確認できます
- `4択クイズ` は、候補が4つそろうカードだけを自動で使うので、少ないデッキでも2択や3択にはなりません
- `復習待ち / 直近ミス / あいまい多め / 伸び悩み` の4つの弱点スマートリストから出題元を選べます
- `AIで高精度生成` を使うと、選択中のデッキや弱点リストから AI 問題案を作って、そのまま開始またはカード候補化できます

## AI 機能を有効にする

Gemini を使わない場合は、追加設定なしで今まで通りローカル専用機能だけで使えます。AI を使いたいときだけ、Vercel に次を追加してください。

- `GEMINI_API_KEY`
- `AI_PROVIDER=gemini`
- `AI_MODEL=gemini-2.5-flash-lite`
- 任意で `AI_ENABLED=true`

補足:

- 無料枠を前提にした実装です
- AI は `PDF取り込み / 過去問参照 / 小テスト生成` の明示操作時だけ使います
- Gemini の無料 tier は Google 公式で `Used to improve our products: Yes` です
- 無料枠や設定不足で AI が使えないときは、有料へ勝手に切り替えずローカル処理へ戻ります

## ホーム画面について

- `ホーム` は科目別のデッキ一覧が主役で、医学 / 英語 / 共有デッキを上から見渡せます
- 各デッキ行から `学習 / 編集 / 資料追加` を直接開けます
- `資料追加` の先頭には `クイック追加` があり、思いついた内容をそのデッキ文脈のまま連続で保存できます
- `編集` を押すと、そのデッキ専用の編集ワークスペースが開き、名前変更やカード編集をまとめて行えます
- `資料追加` では `カード追加 / PDF取り込み / 過去問参照` を、そのデッキに合わせて始められます
- 履歴は下部のコンパクトカードにまとめ、必要なときだけ `詳しく見る` で展開します
- 画面上部の `機能検索` から、`PDF / 過去問 / 共有 / バックアップ / 初期化` のような機能を直接探して開けます

## カード作成の時短機能について

- `作成 > クイック追加` では、`問題 / 答え / 画像 / タグ` だけで素早くカードを追加できます
- `保存して次へ` を使うと、同じデッキ文脈を保ったまま連続でカードを増やせます
- `作成 > 一括入力` では、`Q: / A:`、箇条書き、番号付きメモを貼ると候補レビューへ送れます
- デッキごとに `既定の分野 / 既定タグ / 問題文テンプレ / 答えテンプレ / カードスタイル` を持てます
- 詳細入力フォームや編集ワークスペースでも、これらのデフォルトが新規カード作成時の初期値として使われます

## 画像付きカードについて

- `カード作成` と `編集ワークスペース` で、表裏それぞれに画像を追加できます
- 1面につき最大4枚、1枚あたり8MBまでです
- 文字なしで、画像だけのカードも作れます
- 学習画面では表画像・裏画像がその面に応じて表示され、タップで拡大できます
- ライブラリとデッキ詳細では、画像枚数と先頭サムネイルで画像付きカードを見分けられます

## 設定について

- `設定` タブに、アカウント保存、ガイド再表示、JSON エクスポート / インポート、初期化をまとめています
- `アカウント保存` では、同じ Supabase ログインで自動バックアップと復元ポイント管理ができます
- 自動バックアップはローカル優先のまま動き、ログイン中だけ 15 秒後にクラウドへ保存します
- バックアップ対象は、デッキ、カード、画像、学習履歴、設定、現在見えている共有デッキ内容です
- 初期化は `サンプルを読み込む` の確認入力に加えて、最終確認ダイアログでも承認が必要です
- 誤操作を避けるため、日常的な作成導線から危険操作を分離しています

## アカウント保存について

- `設定 > アカウント保存` から使えます
- 共有と同じ Supabase アカウントを使い、Google / Apple / メールのどれでも同じ保存先を使います
- 初回でも、そのままアカウント作成とログインが行われます
- 自動バックアップは `最新の自動バックアップ 1件` を保ち、さらに `手動保存 / 復元前の安全退避` を復元ポイントとして最大5件保持します
- `この端末に復元` を押すと、先に現在データを安全退避してから上書き復元します
- 未ログインでも、従来どおり `JSONバックアップ` は使えます
- 共有デッキもスナップショットに含みますが、復元後は現在の共有権限を優先し、アクセス権を失ったものはローカルコピーとして残ります

## 共有機能を有効にする

共有なしなら、追加設定なしでそのままローカル専用アプリとして使えます。共有デッキを使いたいときだけ、Supabase を接続してください。

1. Supabase プロジェクトを作る
2. Supabase の SQL Editor で [supabase-schema.sql](/Users/hattoriyuusuke/Documents/New%20project/supabase-schema.sql) を実行する
3. Google ログインを使うなら、Supabase Auth の Providers で `Google` を有効化する
   - Google Cloud / Google Auth Platform で Web OAuth client を作成する
   - Supabase の Google provider 画面に必要情報を入れる
   - callback URL は Supabase の provider 設定画面に表示されるものをそのまま使う
4. Apple ログインを使うなら、Supabase Auth の Providers で `Apple` を有効化する
   - Apple Developer で Services ID / Key を作成する
   - Supabase の Apple provider 画面に必要情報を入れる
   - callback URL は Supabase の provider 設定画面に表示されるものをそのまま使う
5. Supabase Auth の URL 設定で、`Site URL` と `Redirect URLs` に本番 URL を追加する
   例: `https://pulse-recall.vercel.app`
6. Vercel の Environment Variables に次を追加する
   - `SUPABASE_URL`
   - `SUPABASE_ANON_KEY`
   - 任意: `AUTH_GOOGLE_ENABLED`
   - 任意: `AUTH_APPLE_ENABLED`
   - 任意: `AUTH_MAGIC_LINK_ENABLED`
7. Vercel を再デプロイする
8. iPhone または Safari で `Google / Apple / メール` の3方式を確認する

共有フローのポイント:

- Supabase 未設定でも、まずは `共有リンクで複製` が使えます
- `画像付きデッキ` は通常の `?deckshare=` ではなく、共有画面の `画像込みコピーリンク` から `?copy=` の複製リンクを発行します
- 受け取った相手はリンクを開いて、自分のローカルデッキとして追加できます
- 普段の学習はログイン不要です
- 共有リンクを作る/開くタイミングだけログインします
- 設定画面では `Google / Apple / メール` の3方式をまとめ、共有画面はログイン状態の確認と設定への誘導に寄せています
- 共有されるのはデッキ内容で、学習進捗はユーザーごとに分離されます
- 共有リンクの参加は owner 承認制です
- 参加申請時に `閲覧のみ / 編集可` を選べます
- owner は共有後もメンバーのロール変更、詳細権限変更、リンク再発行、共有終了ができます
- 共有中の通常編集は `デッキ情報ロック / カード単位ロック` で保護され、同時編集の上書き事故を減らします
- `共有通知` に、承認待ち、権限変更、共有更新が表示されます
- `更新履歴` で、カード追加・更新・削除、ロール変更、申請承認/拒否などを追えます
- 共有から外れたデッキは、その端末ではローカルコピーとして残ります

Supabase を更新したときは、最新の [supabase-schema.sql](/Users/hattoriyuusuke/Documents/New%20project/supabase-schema.sql) を SQL Editor で再実行してください。今回の共有強化では、`shared_edit_locks`、`shared_deck_events`、`user_notifications`、`shared_copy_packages`、`deck_members.permissions`、Storage bucket `shared-copy-packages` が追加されています。

Google / Apple ログインの補足:

- フロントでは Supabase JS の OAuth redirect を使っています
- iPhone 互換性を優先して popup ではなく redirect で戻します
- Apple は名前が十分に取れないことがあるため、初回ログイン後に表示名の保存欄が出ることがあります
- Apple の秘密鍵は期限管理とローテーションが必要です

`local-only` のままになる時の確認ポイント:

1. `https://pulse-recall.vercel.app/api/client-config` を開き、`supabaseUrl` と `supabaseAnonKey` が空でないか確認する
2. 空なら、Vercel に `SUPABASE_URL` と `SUPABASE_ANON_KEY` が入っていません
3. 値が入っているのに Google / Apple が失敗するなら、Supabase Auth の provider 有効化と callback / redirect URL を見直す
4. メールログインも使えないなら、`AUTH_MAGIC_LINK_ENABLED` が false になっていないか確認する

失敗しやすい点:

- `SUPABASE_URL` / `SUPABASE_ANON_KEY` を Vercel に入れ忘れている
- Supabase Auth の `Site URL` / `Redirect URLs` に本番 URL を追加していない
- Google provider を ON にしたつもりで、Google Cloud 側の OAuth client と callback が噛み合っていない
- Apple の Services ID / Key / callback URL が一致していない
- provider を Supabase 側で ON にしていないのに、Vercel 側の表示フラグだけ true にしている

公開URLが確定したあとにやると良いこと:

- Google Search Console に登録
- 必要なら独自ドメインを設定
- URL 確定後に `sitemap.xml` を追加
